# Мобильный полноэкранный поиск с табами и «Недавними» — Design

## Контекст

Batch1 п.12 («глобальный поиск — выдвигающийся список по типам») реализован
только для десктопа: `GlobalSearch.tsx`, смонтирован в `DesktopTopBar.tsx`,
показывает live-дропдаун под инпутом, сгруппированный по разделам (Люди /
Сообщества / Объявления / Направления).

На мобилке поиска **фактически нет** — иконка «Поиск» в `MobileHeader.tsx`
(строки 56–63) сейчас просто `<Link to="/ads">`, без какого-либо
поле-ввода/выдачи. Это не редизайн существующей мобильной фичи, а её
первое появление.

Референс — скриншоты ВКонтакте (переданы пользователем текстовым описанием
в этой сессии, без файлов изображений): полноэкранная панель снизу/сверху
с полем+«Отмена», табами-фильтрами по типу, блоком «Недавние» и
результатами, сгруппированными по разделам.

### Ключевая находка при исследовании кода

В проекте уже есть механизм «недавно просмотренного» —
`src/lib/view-history.ts` (`getViewHistory()`/`recordView()`,
localStorage-backed, до 50 записей). Тип `kind` уже включает `"profile"`,
но **ни один роут его не проставляет** — `recordView()` сегодня вызывается
только в `ads.$id.tsx:96` (`kind: "ad"`) и `reviews.$id.tsx` (`kind:
"review"`). Просмотры профилей/сообществ никогда не писались. Отдельного
механизма истории **поисковых запросов** (текстовых строк) в проекте нет.

## Решения, зафиксированные при брейншторме

1. **Источник блока «Недавние» = расширенный `view-history`** (не новая
   сущность «история поисковых запросов», не пустой каркас-заглушка).
   В рамках этой задачи `recordView()` дописывается в `user.$id.tsx` и
   `communities.$id.tsx`, `kind` расширяется значением `"community"`.
2. **Визуальный паттерн = полный тейковер** (`position:fixed; inset:0`),
   собственная framer-motion анимация (как `AdFiltersSheet`), **не** через
   существующий `vaul` `Drawer` (тот занят под bottom-sheet бургер-меню и
   не подходит для полновысотного экрана с активной клавиатурой).
3. **Общая логика поиска выносится в хук**, переиспользуется десктопным
   дропдауном и новым мобильным оверлеем — без дублирования запросов.
4. **Табы фильтруют уже загруженные результаты на клиенте**, не делают
   отдельный запрос на каждый таб (все 4 источника и так грузятся вместе).
5. **Оверлей — локальный UI-стейт, не роут.** Системная кнопка «назад»
   закроет вкладку/приложение, а не только оверлей — честно
   задокументированное ограничение, не regression относительно остальных
   модалок проекта (`CreatePostModal`, `AdFiltersSheet` — то же поведение).

## Архитектура

### A. Расширение `view-history.ts`

`ViewHistoryItem.kind`: `"ad" | "profile" | "review"` →
`"ad" | "profile" | "review" | "community"`. Остальной модуль
(`getViewHistory`, `recordView`, `clearViewHistory`, localStorage-ключ,
CAP=50) не меняется.

**Новые вызовы `recordView()`** (по образцу `ads.$id.tsx:96`):

- `src/routes/user.$id.tsx`, внутри `fetchPublicProfile(id).then((p) => { setProfile(p); ... })`:
  ```ts
  recordView({ id: p.user.slug ?? p.user.id, kind: "profile", title: p.user.name, thumb: p.user.avatar });
  ```
- `src/routes/communities.$id.tsx`, внутри `fetchCommunity(id).then((c) => { setCommunity(c); ... })`:
  ```ts
  recordView({ id: c.id, kind: "community", title: c.name, thumb: c.avatarImage });
  ```

Если `getViewHistory()` возвращает `[]` (новый пользователь/чистый
localStorage) — блок «Недавние» **не рендерится вовсе** (ни заголовка, ни
плейсхолдера). Никаких выдуманных данных.

### B. Общий хук поиска — `src/lib/hooks/useGlobalSearch.ts` (новый файл)

Извлекает debounce+fetch-логику, сейчас живущую внутри `GlobalSearch.tsx`
(строки 92–126): `searchUsers`, `fetchCommunities`,
`fetchListings({ q, perPage })`, `fetchListingCategories()` (с
клиентской фильтрацией по имени, как сейчас).

```ts
export interface SearchResults {
  users: User[];
  communities: Community[];
  ads: Ad[];
  categories: Category[];
}
export function useGlobalSearch(
  query: string,
  opts?: { perPage?: number },
): { results: SearchResults; loading: boolean };
```

Сигнатура и поведение (MIN_QUERY_LENGTH=2, DEBOUNCE_MS=300,
request-id guard против гонок устаревших ответов) переносятся как есть.
`perPage` по умолчанию сохраняет текущие десктопные лимиты (4 users / 4
communities / 5 ads / 5 categories); мобильный оверлей передаёт больший
`perPage` при активном (не «Все») табе — см. раздел D.

`GlobalSearch.tsx` переписывается на использование этого хука (без
изменения видимого поведения десктопного дропдауна — чистый рефактор
этой части).

### C. Общие презентационные блоки — `src/components/layout/search/SearchResultRow.tsx` (новый файл)

`SearchGroup` и `ResultRow` (сейчас module-private внутри
`GlobalSearch.tsx`, строки 21–87) выносятся сюда с `export`, без
изменения вёрстки/пропсов. Импортируются и десктопным дропдауном, и
новым мобильным оверлеем — карточка результата выглядит идентично в
обоих местах.

### D. `src/components/layout/MobileSearchOverlay.tsx` (новый файл)

```ts
interface Props {
  open: boolean;
  onClose: () => void;
}
```

Полноэкранная панель, `position:fixed; inset:0; z-50`, `height:
100dvh` (не `100vh` — см. раздел «Клавиатура» ниже), `flex flex-col`.
Framer-motion: fade + небольшой translateY при монтировании/размонтировании
(как `AdFiltersSheet`), `AnimatePresence` — компонент рендерится только
при `open`.

**Структура (сверху вниз):**

1. **Шапка** (`shrink-0`, `padding-top: var(--safe-top)`): `<input
   type="search" autoFocus>` + кнопка «Отмена» (`onClick={onClose}`,
   сбрасывает локальный `query`/`activeTab` state оверлея).
2. **Табы** (`shrink-0`, горизонтальный ряд): «Все / Люди / Сообщества /
   Объявления / Направления» — паттерн активной/неактивной кнопки как в
   `AdFilters.tsx` (`background: var(--accent-soft)` для активного).
   Локальный state `activeTab`, чисто клиентская фильтрация —
   переключение таба не создаёт новый HTTP-запрос.
3. **Тело** (`flex-1 overflow-y-auto`):
   - `query.trim().length === 0` → блок «Недавние», если
     `getViewHistory().length > 0`: карточки `ResultRow`-подобного вида
     (thumb/title), ссылка строится по `kind` (`ad→/ads/$id`,
     `profile→/user/$id`, `review→/reviews/$id`,
     `community→/communities/$id`). Пусто — тело пустое.
   - Есть запрос → `useGlobalSearch(query, { perPage: activeTab ===
     "all" ? undefined : 20 })`. При `activeTab === "all"` — те же 4
     `SearchGroup`, что на десктопе (Направления/Люди/Сообщества/
     Объявления), с текущими лимитами. При конкретном табе — только
     соответствующая секция, без заголовка группы (она и так единственная),
     до 20 результатов.
   - `loading`/`hasAny === false` — те же текстовые состояния, что в
     `GlobalSearch.tsx` («Ищем…» / «Ничего не найдено»).

**Открытие (`MobileHeader.tsx`, строки 56–63):** `<Link to="/ads">` →
`<button aria-label="Поиск" onClick={() => setSearchOpen(true)}>`, новый
локальный `useState<boolean>` в `MobileHeader`. `<MobileSearchOverlay
open={searchOpen} onClose={() => setSearchOpen(false)} />` рендерится
рядом (вне `<header>`, `position:fixed` не зависит от родителя).

## Без дёргания layout (пересекается с принципом из недавнего фикса профильного меню)

- Оверлей `position:fixed` — вне потока документа, монтирование/
  размонтирование не толкает ничего на странице позади.
- `document.body.style.overflow = "hidden"` на время `open` (паттерн уже
  есть в `CreatePostModal.tsx`) — фон не скроллится под оверлеем.
- Анимация (opacity/translateY) — только на контейнере оверлея, не на
  элементах шапки/табов — без layout shift.

## Клавиатура не перекрывает результаты

- `height: 100dvh` (динамический viewport height), не `100vh` — уже
  установленный в проекте паттерн именно для мобильной клавиатуры
  (комментарий в `AppLayout.tsx`: «100dvh keeps the shell stable on mobile
  Safari/Chrome»).
- `flex flex-col h-[100dvh]`: шапка+табы `shrink-0`, тело `flex-1
  overflow-y-auto`. При появлении клавиатуры `dvh` пересчитывается
  браузером, `flex-1`-тело сжимается и остаётся полностью
  прокручиваемым — не проваливается под клавиатуру.
- `<input type="search" autoFocus>` — клавиатура появляется сразу при
  открытии, как в референсе.

## «Отмена» / закрытие

- Кнопка «Отмена» → `onClose()` → `setSearchOpen(false)` в
  `MobileHeader` → размонтирование оверлея. Экран позади не менялся всё
  это время (оверлей — не роут), поэтому «возврат к предыдущему экрану»
  происходит автоматически, без явной навигации.
- **Честное ограничение:** системная кнопка/жест «назад» закроет
  вкладку/приложение целиком, а не только оверлей — это НЕ regression:
  то же поведение у всех остальных модалок проекта (`CreatePostModal`,
  `AdFiltersSheet`), не уникальная проблема этой фичи.

## Тестирование / приёмка

- `tsc --noEmit` чист.
- Live-проверка на demo-стенде, вьюпорты 360/390/430:
  - Клик по «Поиск» в `MobileHeader` → полноэкранный оверлей открывается
    без прыжка контента позади, клавиатура появляется сразу
    (`autoFocus`).
  - Пустой запрос + пустая `view-history` → тело пустое, «Недавние» не
    отображается.
  - Пустой запрос + непустая `view-history` (после расширения на
    profile/community) → блок «Недавние» показывает реальные записи с
    правильными ссылками по `kind`.
  - Ввод запроса (≥2 символа) → результаты как на десктопе, сгруппированные
    по разделам при табе «Все».
  - Переключение таба на «Люди»/«Объявления»/etc. → мгновенно (без
    нового запроса) фильтрует уже загруженные результаты, показывает
    расширенный список только этого типа.
  - Клавиатура открыта → последний результат в списке виден и
    доскроллиается, не обрезан.
  - «Отмена» → оверлей закрывается, лента/страница позади в точности та
    же, что была до открытия (не перезагружена, скролл-позиция сохранена).
  - Десктопный дропдаун (`GlobalSearch.tsx`) — визуально и функционально
    не изменился после рефакторинга на `useGlobalSearch`.

## Не входит в эту фазу

- История **поисковых запросов** (текстовых строк) — сознательно не
  строится; «Недавние» показывает просмотренные сущности, не введённые
  запросы (см. «Решения» п.1).
- Синхронизация оверлея с роутером/browser history — системная кнопка
  «назад» не закрывает оверлей отдельно от страницы (см. «Честное
  ограничение» выше); если это станет проблемой в реальном использовании
  — отдельная задача.
- Замена десктопного UX — `DesktopTopBar`/`GlobalSearch` остаются
  дропдауном, полноэкранный оверлей — только для `lg:hidden`
  (мобильного) брейкпоинта.
