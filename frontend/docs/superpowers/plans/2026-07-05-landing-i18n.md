# Landing i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Локализовать лендинг (`routes/index.tsx`) на ru/en/zh через i18next и вернуть LanguageSwitcher на топ-нав.

**Architecture:** Добавить поддерево `landing.*` в три локали (ru — источник схемы, en/zh типизированы как `typeof ru` → tsc гарантирует полноту). Переписать `index.tsx`: весь видимый текст → `t("landing.…")`; data-массивы хранят иконку+route+`key`, текст берётся при рендере.

**Tech Stack:** React 18, i18next / react-i18next, TanStack Router, TypeScript strict.

## Global Constraints

- Работать строго в `frontend/`; backend НЕ трогать.
- `npx tsc --noEmit` чист после финальной задачи (промежуточно ru-only ломает en/zh — это ожидаемо, см. порядок задач).
- Ветка `feature/landing-i18n` (создана). Деплой на `neeklo`.
- SEO `head()` мета — русская, НЕ трогать.
- Бренд: ru «МоДелизМ», en/zh «Modelizm». Категории переводятся. Валюта ₽ + `toLocaleString("ru-RU")` без изменений.
- `CONDITION_COLOR` (сравнение с demo-строкой «Новое»), alt-тексты из `ad.title`, все inline-стили/motion — НЕ трогать.
