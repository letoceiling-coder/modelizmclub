|---------|------------|-------------|
| **Backend (Этап 1)** | ~92% | Ядро, лента, сообщества, модерация — готовы; OAuth и жалобы пользователя — нет |
| **Backend (Этап 2)** | ~68% | Чат, объявления, биллинг — готовы; ИИ, геймификация — нет |
| **Backend (Этап 3)** | ~8% | Только схема БД и базовый admin-дашборд |
| **Frontend (все этапы)** | ~70% | UI широкий, но оплата/OAuth/часть экранов — заглушки; demo mode на localhost |
| **Инфраструктура** | ~75% | VDS, PostgreSQL, Redis, S3, Reverb — есть; Horizon, CI/CD, нагрузочные тесты — нет |

**Ответ на вопрос «Всё ли реализовано в backend для завершения проекта?» — НЕТ.**

Backend закрывает большую часть Этапа 1 и значительную часть Этапа 2, но **не готов к финальной сдаче по ТЗ**. Для полного закрытия backend нужно как минимум: OAuth (VK/Яндекс), API жалоб, ИИ-помощник объявлений, геймификация (таблицы + API), интеграции доставки, чат-бот поддержки, расширенная аналитика, Horizon/очереди, 2FA для админов.

## 10. Доработки backend (02.07.2026)

Реализовано после аудита — расширение фильтрации API и закрытие части пробелов.

### Расширенная фильтрация и сортировка списков

| Эндпоинт | Добавлено |
|----------|-----------|
| `GET /feed` | `q`, `hashtag`, `community_id`, `has_media`, `date_from`, `date_to`, `sort=new\|popular\|discussed\|viewed\|oldest` |
| `GET /listings` | `price_min`, `price_max` (₽), `category_ids[]`, `delivery_method`, `has_media`, `sort=newest\|oldest\|price_asc\|price_desc\|popular\|favorites` |
| `GET /users/me/listings` | `status`, `q`, `sort` |
| `GET /users/search` | `city_id`, `interest`, `sort=newest\|name\|popular\|rating` |
| `GET /communities` | `sort=popular\|newest\|name` |

Во всех списках `per_page` ограничен максимумом 50.

### Новые эндпоинты

| Метод | Путь | Назначение |
|-------|------|------------|
| POST | `/reports` | Жалоба пользователя (post/listing/comment/user) с защитой от дублей и жалоб на свой контент |
| GET | `/admin/reports/{id}` | Карточка жалобы (admin/moderator) |
| PATCH | `/admin/reports/{id}` | Обработка жалобы (`reviewing/resolved/rejected/dismissed`) |
| POST/DELETE | `/comments/{uuid}/react` | Реакции на комментарии (таблица `comment_reactions`) |
| POST/DELETE | `/listings/{uuid}/favorite` | Избранное объявлений (`listing_favorites`) |
| GET | `/users/me/favorites` | Список избранных объявлений |
| GET | `/users/me/subscription` | Текущая подписка + план (`is_active`, `days_left`) |
| POST | `/listings/ai-suggest` | ИИ-подсказка категории/описания/тегов |

### ИИ-помощник — «разные варианты» (провайдеры)

`config/listing.php` → `LISTING_AI_PROVIDER`:
- `heuristic` (по умолчанию) — офлайн-сопоставление текста с категориями, без внешних вызовов;
- `openai` — OpenAI-совместимый API (`LISTING_AI_OPENAI_*`) с безопасным откатом на эвристику при любой ошибке.

Архитектура: контракт `ListingSuggestionProvider` + `HeuristicSuggestionProvider` / `OpenAiSuggestionProvider` + `ListingSuggestionManager`.

### Тесты

Добавлены `tests/Feature/ListingFilterAndExtrasTest.php` и `tests/Feature/ReportAndCommentReactionTest.php` (фильтры, избранное, ИИ-подсказка, жалобы, реакции на комментарии). Запуск — в CI/на VPS (локально нет `pdo_sqlite`).

### Осталось (не входило в доработку)

OAuth VK/Яндекс, геймификация (таблицы + API), Horizon/очереди, промокоды user-flow, интеграции доставки, чат-бот поддержки — см. разделы 4.1–4.3.
