## 🔴 Hard blockers (в порядке важности)

### 1. §27 — Заявки на создание Канала / Сообщества
**Самый крупный блокер.** Фронт полностью готов и задеплоен (форма заявки, CTA
«Хочу свой» на страницах, `/settings/spaces`, admin-раздел «Заявки»), но в
реальном режиме флоу мёртв. Нужно:
- **approve → создать сущность → назначить владельца** (обе сущности). Для
  сообщества — **добавить значение `owner` в enum `CommunityMemberRole`**
  (сейчас только `member`/`moderator`), иначе «назначить инициатора
  администратором» невозможно. Текущий `ModerationService::approve` для
  `Community` лишь флипает статус существующей записи и никого не назначает.
- Admin-стек обзора заявок: `GET/POST /admin/communities/applications...`
  (сейчас `community_applications` пишется через `POST /communities/apply`, но
  **никто не читает** — нет роута/контроллера).
- Канал: весь стек новый — таблица+модель `channel_applications`,
  `POST /channels/apply`, admin list/approve/reject, создание `Channel` с
  `owner_id = applicant`.
- Подключить существующий `POST /communities/apply` к фронту (guard «уже
  pending» бэк уже отдаёт).

Аудит первопричин: `docs/communities-channels-ownership-audit.md`.

### 2. §16 — Сброс пароля (P0-баг на бэке)
Новый пароль после сброса не подходит для входа. Требует расследования на
бэкенде — блокирует реальный auth-флоу восстановления доступа.

### 3. §26 — Иконки (кастомные SVG, админка, фаза 2b)
Фронт готов (загрузка/санитизация-превью/перекраска/публикация в demo). Реальный
режим блокирован: нужны `IconAsset`-модель, серверная SVG-санитизация+токенизация,
`POST /media purpose=icon` → IconAsset, `GET/DELETE /admin/icon-assets`, публичный
`GET /icon-overrides`.

## QA-статус фронта (что уже проверено живьём на neeklo)

Собрано и живо-верифицировано в этой сессии (desktop 1280 + mobile 360/390/430,
demo-режим, Playwright):

- **Каталог:** фильтр «Доставка» убран из UI; каталог + фильтры без оверфлоу.
- **Профильное меню (desktop):** hover-открытие/закрытие, Escape, click-outside,
  без флика (root cause — Radix `modal` + phantom-hover, `modal={false}` + guard).
- **Кроп фото профиля:** ползунок масштаба больше не искажает пропорции
  (`maxWidth/maxHeight:none` против Tailwind Preflight).
- **Мобильный полноэкранный поиск:** табы Все/Люди/Сообщества/Объявления/
  Направления, «Недавние» из view-history, переключение табов без нового
  запроса, «Отмена» без прыжка layout. Десктопный дропдаун не изменился.
- **Композер «Создать»:** минимальная кнопка → меню тип (Пост/Видео) → источник
  (профиль/канал у владельца), десктоп-дропдаун + мобильный bottom-sheet;
  авто-открытие файл-пикера; реальная публикация от профиля (uploadMedia→
  createPost→publishPost); канал-ветка через существующий `createChannelPost`.
- **Перетаскивание фото (Avito-style):** dragged-плитка следует за указателем
  (translate 1:1, тень, pointer-events:none), порог подъёма 5px, коммит порядка
  на отпускании — проверено реальным mouse-drag (reorder 0→2 корректен).
- **Заявки на создание:** CTA «Хочу свой» открывает форму нужной сущности,
  `/settings/spaces` две карточки, admin «Заявки» список с бейджами +
  оптимистичный approve/reject.

**Финальный sweep (2026-07-15):** 12 маршрутов на десктопе — корректные
заголовки, ноль горизонтального оверфлоу, ноль новых console-ошибок (кроме
известных pre-existing admin-CORS на `/admin`, real-API не demo-mock). 7 ключевых
маршрутов на 360/390/430 — ноль оверфлоу. `tsc --noEmit` чист. local == remote
== server @ `426bca9`.

**Известные ограничения проверки (честно):** реальный (не demo) режим на neeklo
недоступен без бэкенд-эндпоинтов и реальных креденшлов; ветки «Мой канал» (нет
demo-канала с `isOwner=true`) и серверная персистентность заявок проверены только
инспекцией кода в per-task ревью. Пре-существующие admin-CORS ошибки на `/admin` —
это real-API вызовы, не demo-mock, не регрессия этой сессии.
