# Карта личного кабинета продавца — что есть / чего не хватает

> Аудит по состоянию репозитория (ветка `feature/audit-round3-fixes`).
> Только отчёт, изменений кода нет.
> Стек: backend — Laravel (модульный, `backend/app/Modules/*`), frontend — TanStack Start / React (`frontend/src`).

## TL;DR (главное в одном абзаце)

Схема БД под полноценный кабинет продавца **уже почти вся есть** (объявления со статусами, счётчики просмотров/избранного, подписки, платежи, промо-размещения, правила ценообразования, промокоды, бонусы). **Просмотры реально трекаются.** **Реальные платёжные интеграции написаны** (ВТБ-эквайринг + ЮKassa + stub) — но **выключены** (нет ключей в env) и **фронт их вообще не вызывает**. Чего физически нет: **дохода** (монетизация не запущена — платными сейчас может быть только подписка, да и то оплата не подключена), **дашборда со статистикой** (есть только 4-плиточная сводка в «Мои объявления», где «Сумма» — это сумма ценников, а не выручка), **истории просмотров по дням** (хранится только общий счётчик, графики строить не из чего), и ряда разделов настроек продавца (смена пароля из кабинета, сохранённые карты, 2FA, серверное хранение реквизитов).

## Сводная приоритизация (что «добавить» реально значит)

1. **Активировать оплату** (высокий эффект, малый объём кода на бэке): env-ключи ВТБ/ЮKassa + фронтовый флоу `POST /payments` → checkout. Без этого нет ни подписок за деньги, ни платных размещений, ни «дохода».
2. **Смена пароля из кабинета** и **серверные реквизиты** — закрыть демо-заглушки настоящими эндпоинтами (`POST /account/change-password`, `GET/PUT /account/requisites`).
3. **Платные размещения/бусты** — дописать флоу оплаты для `listing`/`promotion` (схема БД уже есть, `CreatePaymentController` пока только подписки).
4. **Дашборд продавца** — `GET /users/me/stats` + экран; исправить «Сумма» (сейчас = сумма ценников, не выручка).
5. **Динамика просмотров** (если нужны графики) — временной ряд по дням (сейчас только суммарный счётчик).
6. **2FA / сохранённые карты** — по мере зрелости (требуют отдельных решений по безопасности/PCI).
